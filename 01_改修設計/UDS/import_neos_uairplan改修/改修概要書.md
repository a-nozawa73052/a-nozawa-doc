# 改修概要書：neos_uairplan 取込アーキテクチャ変更

作成日: 2026-06-05  
対象システム: UDS (uds-web)

---

## 1. 改修背景・目的

### 背景

現行の `import_neos_uairplan` は以下の3-Lambda 構成で動作している。

1. `split-neos-uairplan-tsv`: 1,000行ずつ分割 → `split/` へアップロード
2. `batch-transfer-neos-uairplan-tsv`: 10分ごとに `split/` の最古ファイル1件を `ready/` へ転送
3. `import_neos_uairplan`: `ready/` への S3 PUT をトリガーに UDS DB を更新 → SGS へファイルコピー

### 課題

| 課題 | 内容 |
|-----|------|
| 処理遅延 | ファイル到着から処理完了まで最大「ファイル数 × 10分」かかる。190,000行の場合、分割後190ファイル × 10分 = 約1,900分（31時間超） |
| 無駄な処理 | 190,000行のうち UDS に存在する契約は約2,000件（約1%）のみ。全行を read/transfer していた |
| 1ファイルずつしか処理できない | `batch-transfer` が1回の実行で1ファイルのみ転送するため、大量ファイル到着時に渋滞が発生する |

### 目的

- 10分ごとの DB 更新件数を **1,000件以内** に制限する
- 190,000行規模のファイルが到着しても **処理遅延を最小化する**

---

## 2. 改修対象 Lambda 一覧

| Lambda | 変更種別 | 主な変更内容 |
|--------|---------|-------------|
| `split-neos-uairplan-tsv` | **改修・改名** | Lambda名を `preprocess-neos-uairplan-tsv` に変更。SGSへの元ファイルコピー追加、contract_cd バルクフィルタリング追加、一次テーブルへの INSERT 追加。ファイル分割処理は廃止 |
| `batch-transfer-neos-uairplan-tsv` | **廃止** | preprocess が機能を吸収するため廃止 |
| `import_neos_uairplan` | **改修** | トリガーを S3 PUT → EventBridge（10分ごと）に変更。S3ファイル読込を一次テーブル読込に変更。SGSコピー処理を削除。1,000行単位処理・処理後 DELETE を追加 |

### 新規作成物

| 種別 | 名称 | 内容 |
|-----|------|------|
| DBテーブル | `work_imported_neos_uairplan` | UDS RDS（uds DB）に追加する一次テーブル。preprocess から import へのデータ受け渡しバッファ |

## 3. 変更前後フロー比較

### 変更前

```
UNIS
  ↓ neos_uairplan_yyyymmddhhiiss.tsv
S3: {env}-udsapi-neos-uairplan（入力バケット）
  ↓ S3トリガー
[1] split-neos-uairplan-tsv
    └─ 1,000行ずつ分割 → {env}-udsapi-split-neos-uairplan-tsv/split/ へアップロード
  ↓ EventBridge（10分ごと）
[2] batch-transfer-neos-uairplan-tsv
    └─ split/ の最古ファイル 1件 → ready/ へコピー・元ファイル削除
  ↓ S3トリガー（ready/ PUT）
[3] import_neos_uairplan
    ├─ UDS DB（RDS）各テーブルを更新
    └─ SGS S3バケットへファイルコピー
```

### 変更後

```
UNIS
  ↓ neos_uairplan_yyyymmddhhiiss.tsv
S3: {env}-udsapi-neos-uairplan（入力バケット）
  ↓ S3トリガー
[1] preprocess-neos-uairplan-tsv（旧: split-neos-uairplan-tsv）
    ├─ ① SGS S3バケットへ元ファイルをコピー
    ├─ ② S3ストリームで1,000行ずつ読込
    │      └─ バルク SELECT（IN句）で UDS.contracts.contract_cd と照合
    └─ ③ 一致した行のみ work_imported_neos_uairplan テーブルへ INSERT
  ↓ EventBridge（10分ごと）
[2] import_neos_uairplan（改修）
    ├─ work_imported_neos_uairplan から古い順（id ASC）に最大1,000行取得
    ├─ UDS DB（RDS）各テーブルを更新（現行ロジックと同等）
    └─ 処理済み行を work_imported_neos_uairplan から DELETE
```

## 4. 期待効果

| 項目 | 変更前 | 変更後 |
|-----|-------|-------|
| 190,000行ファイルの staging 格納時間 | 約1,900分（31時間超） | 約30秒以内 |
| UDS契約分（2,000件）の処理完了時間 | 約1,900分以上 | 約20分（2バッチ） |
| 10分あたりの DB 更新件数上限 | 制限なし | **1,000件** |
| RDS への同時接続数 | 最大ファイル数分（並列処理時） | 1（逐次処理） |
| Lambda 総本数 | 3本 | **2本** |

## 2. preprocess-neos-uairplan-tsv 設計（改修・改名）

### 変更概要

| 項目 | 変更前（split-neos-uairplan-tsv） | 変更後（preprocess-neos-uairplan-tsv） |
|-----|--------------------------------|---------------------------------------|
| Lambda名 | `{env}-split-neos-uairplan-tsv` | `{env}-preprocess-neos-uairplan-tsv` |
| トリガー | S3 PUT（入力バケット） | 同左（変更なし） |
| SGS コピー | なし | **追加**（元ファイルを先頭でコピー。コピー先 Key には `key.split('/').pop()` でファイル名のみを使用。現行の `import_neos_uairplan` は `ready/{filename}` をトリガーとするため `.pop()` でパス除去が必要だったが、preprocess は入力バケット直下の `{filename}` をトリガーとするため `'/'` を含まず結果は同じ） |
| ファイル分割 | 1,000行ずつ split/ へアップロード | **廃止** |
| contract_cd フィルタ | なし | **追加**（バルク SELECT で照合） |
| Staging INSERT | なし | **追加** |

### 処理フロー

```
S3 PUT イベント受信（入力バケット）
  ↓
[1] S3 から元ファイルを取得
  ↓
[2] 元ファイルを SGS バケットへコピー
    └─ コピー先: s3://{SGS_BUCKET}/{ファイル名}
    └─ Key には key.split('/').pop() でファイル名のみを使用する
       （現行の import_neos_uairplan は ready/{filename} をトリガーとするため .pop() でパス除去が必要だったが、
         preprocess は入力バケット直下の {filename} をトリガーとするため '/' を含まない。
         結果は同じになるが、移植元コードの意図を明示するため記載する）
    └─ ※ コピー失敗時は処理中止・Google Chat 通知
  ↓
[3] ファイルをストリームで 1,000行ずつ読込
    ├─ タブ区切りで42列に分割
    └─ バリデーション（現行のチェックロジックを流用）
       ├─ 列数チェック（42列であること）
       ├─ NOT NULL チェック（contract_cd, contract_status_cd 等）
       ├─ 文字数チェック（各カラムの maxLength 以内）
       └─ 日時形式チェック（datetime カラム）
  ↓
[4] 1,000行分の contract_cd でバルク SELECT
    └─ SELECT contract_cd FROM contracts
         WHERE contract_cd IN (?, ?, ...) AND deleted_at IS NULL
    └─ ヒットした contract_cd のセットを取得
  ↓
[5] ヒットした行のみ work_imported_neos_uairplan へ INSERT
    └─ 全42カラム + source_file_name（元ファイル名）
    └─ created_at は DB の CURRENT_TIMESTAMP
  ↓
[3]〜[5] をファイル末尾まで繰り返す
  ↓
[6] 入力バケットから元ファイルを削除
```

### エラーハンドリング

| エラー種別 | 対応 |
|-----------|------|
| バリデーションエラー | エラー行をスキップ or 処理中止（現行と同様） |
| SGS コピー失敗 | 処理中止、Google Chat 通知、入力ファイルは保持（次回リトライ） |
| DB 接続エラー | Google Chat 通知、入力ファイルは保持（Lambda の自動リトライ） |
| INSERT 失敗 | トランザクションロールバック後、Google Chat 通知 |

## 3. import_neos_uairplan 設計（改修）

### 変更概要

| 項目 | 変更前 | 変更後 |
|-----|-------|-------|
| トリガー | S3 PUT（ready/ フォルダ） | **EventBridge**（`cron(0/10 * * * ? *)`） |
| データ取得元 | S3 TSV ファイル | **work_imported_neos_uairplan** テーブル |
| SGS コピー | あり（ファイルを SGS バケットへコピー） | **廃止**（preprocess 側に移動） |
| TSV バリデーション | あり | **廃止**（preprocess 側で実施済み） |
| 1回の処理件数 | ファイル1本分（最大1,000行） | 最大1,000行（Staging から取得） |
| 処理後の後処理 | なし | **Staging レコードを DELETE** |

### 処理フロー

```
EventBridge トリガー（10分ごと）
  ↓
[1] work_imported_neos_uairplan から古い順に最大1,000行取得
    └─ SELECT * FROM work_imported_neos_uairplan ORDER BY id ASC LIMIT 1000
  ↓
[2] 取得件数が 0件 → 処理終了（ログ出力のみ）
  ↓
[3] 各行ごとに以下をトランザクション内で実行
    ├─ contracts テーブル UPDATE
    │    └─ contract_cd が一致するレコードを更新
    │    └─ 更新対象外: customer_chain_cd, customer_chain_name
    ├─ contract_com_devices テーブル UPDATE（該当する場合）
    │    └─ neos_item_cd, neos_item_name, com_ssid2_enable を更新
    ├─ contract_mdm_devices テーブル UPDATE（該当する場合）
    │    └─ neos_item_cd, neos_item_name, com_ssid2_name, com_ssid2_pass を更新
    ├─ contract_mdm2_devices テーブル UPDATE（該当する場合）
    │    └─ 複数契約対応（option1〜5 含む）
    ├─ serials テーブル ステータス更新
    │    └─ 契約ステータスの変化を検知して端末ステータスを遷移
    ├─ shipping_requests テーブル UPDATE（該当する場合）
    ├─ direct_shipments テーブル UPDATE（該当する場合）
    ├─ pppoe_request_contracts 登録／削除（PPPoE v4/v6 対応）
    ├─ Arteria SQS へキュー投入（PPPoE 設定）
    ├─ UX SQS へキュー投入（UTVOP30_n01〜n04 の場合）
    ├─ Google Maps API で GPS 座標取得・contract_uphone_locations UPDATE（U-Phone の場合）
    └─ **work_imported_neos_uairplan から該当行を DELETE**（処理後）
  ↓
[4] 処理完了ログ出力
```

### 削除する処理

| 処理 | 理由 |
|-----|------|
| S3 TSV ファイル読込 | Staging テーブルからの読込に変更 |
| SGS S3 バケットへのコピー | preprocess 側に移動 |
| TSV バリデーション（列数・NULL・文字数チェック） | preprocess 側で実施済み |

### Lambda スペック

| 項目 | 変更前 | 変更後 |
|-----|-------|-------|
| トリガー | S3 PUT（ready/ フォルダ） | EventBridge（10分ごと） |
| Runtime | Node.js 16.x | Node.js 16.x（変更なし） |
| Memory | 512 MB | 512 MB（変更なし） |
| Timeout | 900秒 | 900秒（変更なし） |

### 備考

- `id`（AUTO_INCREMENT）の昇順が処理順序を保証する
- `source_file_name` はトレーサビリティ用（どのファイルから来た行かを追跡可能にする）
- `neos_item_cd`, `neos_item_name`, `service_cd`, `service_name` は `import_neos_uairplan` が `device_types` テーブルから都度取得するため、`work_imported_neos_uairplan` には格納しない
- `customer_chain_cd`, `customer_chain_name` は `work_imported_neos_uairplan` に格納するが、`import_neos_uairplan` での `contracts` UPDATE 時は更新対象外とする

## 8. エラー発生時の手動リカバリ手順

### 8-1. preprocess-neos-uairplan-tsv でエラーが発生した場合

**目的**: preprocess が途中で失敗し、`work_imported_neos_uairplan` への INSERT が完了しなかった場合に再処理する。

**手順**:

1. 連携ファイル（`neos_uairplan_yyyymmddhhiiss.tsv`）内の全 `contract_cd` に対して、UDS DB の `contracts.updated_at` を確認する
2. `updated_at` がファイル連携時刻より**前**の `contract_cd` が存在する場合 → 該当ファイルを入力バケットへ再 PUT してpreprocess を再トリガーする

```sql
-- 確認クエリ例（ファイル連携時刻: '2026-06-08 10:00:00' の場合）
SELECT contract_cd, updated_at
FROM contracts
WHERE contract_cd IN ('xxx', 'yyy', ...)
  AND updated_at < '2026-06-08 10:00:00';
```

> **注意**: 再 PUT するとファイル全行が再処理されるが、`contracts` UPDATE は冪等（同一データで上書き）のため、既処理分に対してデータの巻き戻しは発生しない。ただし `work_imported_neos_uairplan` に既にINSERT済みの行が重複する可能性があるため、再 PUT 前に `work_imported_neos_uairplan` の該当行を確認すること。

---

### 8-2. import_neos_uairplan でエラーが発生した場合

**前提**: `import_neos_uairplan` の処理実行中（毎時 0/10/20/30/40/50 分前後）とかぶらないタイミングで作業すること（データの巻き戻しを防ぐため）。

**① まず contracts.updated_at を確認する**

```sql
-- 対象 contract_cd の処理済み状況を確認
SELECT contract_cd, updated_at
FROM contracts
WHERE contract_cd IN (
  SELECT contract_cd FROM work_imported_neos_uairplan
);
```

**② 処理時刻より前（未処理）の場合**

`updated_at` がエラー発生時刻より前 → `import_neos_uairplan` によって未処理のレコード。

| 状況 | 対応 |
|-----|------|
| SQL でデータ修正が可能 | `work_imported_neos_uairplan` の該当行を UPDATE して内容を修正し、次の 10 分バッチで再取り込みされるのを待つ |
| SQL でのデータ修正が不可能 | `work_imported_neos_uairplan` から該当行をエクスポート（記録）したうえで DELETE し、別途手動対応する |

```sql
-- 修正可能な場合: 該当行を UPDATE して再取り込み待ち状態にする例
UPDATE work_imported_neos_uairplan
SET contract_status_cd = '1'  -- 修正内容に応じて変更
WHERE contract_cd = 'xxx';

-- 修正不可の場合: エクスポート後に DELETE
-- ① まず SELECT でエクスポート・記録
SELECT * FROM work_imported_neos_uairplan WHERE contract_cd = 'xxx';
-- ② 確認後に削除
DELETE FROM work_imported_neos_uairplan WHERE contract_cd = 'xxx';
```

**③ 処理時刻より後（処理済み）の場合**

`updated_at` がエラー発生時刻より後 → `import_neos_uairplan` による更新は完了している。対応不要。

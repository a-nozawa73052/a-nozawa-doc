# IoT 階層管理機能 要件書・仕様書

作成日: 2026-03-24
最終更新: 2026-04-07
対象システム: GRS（usen-grs-mini-web）

---

## 1. 背景・目的

### 背景
従来のGRSデータ構造では、SGSで管理しているデータ構造と異なるためにIoTサービスのデータがGRSで処理出来ない状態にある。
GRSのデータをSGSの階層構造に合わせ、階層構造に合わせた画面を新規作成することによって、GRS起点でのIoTサービスの処理を可能とする。

### 目的
- 本社 → DX企業 → 店舗 の3階層をUI上で管理できるようにする
- 本社・店舗単位でUMIDアカウントを手動で作成・紐づけできるようにする

### 適用範囲
**IoT依頼のみ**（GR商材は対象外）

---

## 2. 階層構造

```
本社（例: ゼンショー）
  └─ DX企業（例: すき家、はま寿司）
       └─ 店舗（例: すき家 渋谷店）
            └─ UMIDアカウント（複数可）
```

### 用語定義

| 新用語 | 旧用語（設計議論時） | SGS対応テーブル | 説明 |
|--------|------------------|----------------|------|
| 本社 | 企業 / ブランド / グループ | companies | 企業グループの最上位。例: ゼンショー |
| DX企業 | DX企業(グループ) / SG | service_groups相当 | 本社配下のグループ単位。例: すき家、はま寿司 |
| 店舗 | 支店 | service_group_customers | 実際の店舗。例: すき家 渋谷店 |

### 既存mock UI との対応

| 既存mock表記 | 新機能での表記 |
|------------|-------------|
| 企業 | 本社 |
| SG（サービスグループ） | DX企業 |
| グループCD / サービスグループCD | DX企業CD |
| 店舗 | 店舗（変更なし） |

## 3. システム要件

| 項目 | 内容 |
|------|------|
| ベースUI | `grs-mini-web` の `/mock/hierarchy-tree.vue` |
| データ保存先 | SGS RDS（GRS DBには保存しない） |
| 書き込み方式 | GRS → SGS RDBへ直接POST |
| GRS側での更新DB | `t_dx_company_cd_numbering`（本社CD発番用）のみ |
| 承認フロー | なし |

## 4. 機能要件

### 4.1 検索機能

検索種別によって結果の表示フローが異なる。

| 検索種別（タブ） | 種類 | 結果フロー |
|----------------|------|----------|
| 本社CD | CD完全一致 | → 直接ツリー表示（本社ノードをハイライト） |
| DX企業CD | CD完全一致 | → 直接ツリー表示（該当DX企業ノードをハイライト） |
| 設置先CD | CD完全一致 | → 直接ツリー表示（該当店舗×サービス行をハイライト） |
| 本社名 | 名前LIKE検索 | → 候補一覧 → 1社選択 → ツリー表示（本社ノードをハイライト） |
| DX企業名 | 名前LIKE検索 | → 候補一覧 → 1社選択 → ツリー表示（該当DX企業ノードをハイライト） |
| 設置名 | 名前LIKE検索 | → 候補一覧 → 1社選択 → ツリー表示（該当店舗×サービス行をハイライト） |

**制約: ツリーは必ず1本社の状態で表示する。フルツリー（全ノード）を表示した上で、検索キーに一致したノードを自動選択・ハイライトする。**

#### 名前検索のフロー

```
① 名前系キーワードを入力して検索
② ヒットした本社の候補一覧を表示（複数件）
   - 本社名 / 本社CD / 顧客CD
③ 一覧から1本社を選択
④ 選択した本社のフルツリーを表示し、検索キーに一致したノードを自動ハイライト
```

### 4.2 本社管理（新規）

| 機能 | トリガー | 処理 |
|------|---------|------|
| 本社追加 | 画面上部「本社追加」ボタン | companiesテーブルにupsert。本社CDをt_dx_company_cd_numberingで発番 |

**本社追加 入力項目**

| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| 本社名 | ✅ | string | 例: ゼンショー |
| UNIS顧客CD | ✅ | char(9) | 本社管理店舗のUNIS顧客CD |
| サービスCD | ✅（newモードのみ） | char(3) | neos_contracts確認用。入力値でUMID自動検出。addGroupモードでは非表示 |

### 4.3 DX企業管理（新規）

| 機能 | トリガー | 処理 |
|------|---------|------|
| DX企業追加 | 本社選択時の右パネル「DX企業追加」ボタン | SGS RDSにPOST |

**DX企業追加 入力項目**

| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| DX企業名 | ✅ | string | 例: すき家、はま寿司 |
| ※所属本社 | 自動設定 | - | ダイアログ表示時に自動セット |
| ※DX企業CD | 自動発番 | char(5) | SGSのservice_group_cdに対応 |

### 4.4 店舗管理（既存機能の継承）

既存の「店舗追加」ダイアログをそのまま使用。DX企業選択時に「店舗追加」ボタンから起動。

**店舗追加 入力項目**

| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| 顧客CD（UNISCD） | ✅ | char(9) | 店舗のUNISCD |
| 店舗名 | | string | |
| 店舗CD | | char(4) | |
| サービスCD | ✅ | char(3) | U40/U60/U62/U90/U91/U93 |
| 権限CD | ✅ | int | 30: 支店管理, 31: 支店シェア |
| サービス開始日 | | date | YYYY-MM-DD |

### 4.5 UMアカウント作成・紐づけ管理（新規）

本社レベル（本社権限）・店舗レベル（支店権限）それぞれで追加可能。
ダイアログは新規作成と既存紐づけを統合した単一フロー。入力したメールアドレスに既存UMIDがあるかバックエンドが判断し、なければ新規作成、あれば既存に紐づける。

**UMアカウント作成・紐づけ 入力項目**

| 項目 | 必須 | 型 | 説明 |
|------|------|-----|------|
| 認証用メールアドレス | ✅ | string | 既存アカウントの場合は自動で紐づけ |
| アカウント名 | ✅ | string | MEMBERSのアカウント表示名 |
| ※サービスCD | 自動設定 | char(3) | ボタン呼び出し元のサービスCDが自動セット（ユーザー入力なし） |

**権限の扱い:**
シェア権限（21: 本社シェア / 31: 支店シェア）は廃止。権限CDはlevelにより自動設定される（本社: 20固定、店舗: 30固定）。UIでの選択は不要。

**ウェルカムメール:**
アカウント作成後、MEMBERS APIがご利用案内メールをアカウントに自動送信する。

**ボタン表示条件:**
同一サービスCD×UNISCDにすでにアカウントが存在する場合、「UMアカウント作成」ボタンを非活性（disabled）にする。

### 5.4 左パネル: ツリービュー

| ノード種別 | アイコン | 色 | 表示内容 |
|----------|--------|-----|---------|
| 本社 | mdi-office-building | blue-darken-2 | 本社名（太字・青） |
| DX企業 | mdi-layers | green-darken-2 | DX企業名 |
| 店舗 | mdi-store | blue-grey | 店舗名 + サービスCDチップ（例: "U40 Camera"） |

**店舗ノードの表示ルール:**
1店舗×1サービス = 1行。複数サービスを持つ店舗は、サービスの数だけ行が作成される。例: 越谷店がU40・U90の2サービスを持つ場合、「越谷店 U40 Camera」と「越谷店 U90 サイネージ」の2行が表示される。

表示項目:

- 配下のDX企業一覧（チップ表示）

ボタン（★新規）:

**本社レベル店舗カード（honshaStores）**

本社に直接紐づく店舗を物理店舗単位で1枚表示。複数サービスが紐づく場合でもカードは1枚。

表示項目:
- 店舗CD / 店舗名 / 管理用店舗UNISCD（1回だけ表示）

## 7. UIフロー

### フロー1: 名前で本社を検索してツリー表示

```
① 検索種別を「本社名」等に切り替え
② キーワードを入力して検索（LIKE検索）
③ 候補一覧に複数本社がヒットして表示される
   （各カードに: 本社名 / 本社CD / 顧客CD / DX企業件数 / 店舗件数）
④ 対象の本社カードの「ツリーを表示」をクリック
⑤ その本社のみのツリー構造が表示される（必ず1本社）
```

### フロー2: CDで本社を検索してツリー表示

```
① 検索種別を「本社CD」等に切り替え
② CDを入力して検索
③ 直接ツリー表示（候補一覧を経由しない）
```

### フロー3: 新規本社作成

```
① 画面上部「本社追加」ボタンをクリック
② 本社追加ダイアログが開く
③ 本社名・UNIS顧客CDを入力
④ 「登録」ボタンをクリック
⑤ SGS RDS companies テーブルにPOST
   t_dx_company_cd_numbering で本社CDを発番
⑥ ダイアログが閉じる
```

### フロー4: DX企業追加

```
① 左ツリーで本社ノードをクリック
② 右パネルに本社情報が表示される
③ 「DX企業追加」ボタンをクリック
④ DX企業追加ダイアログが開く（所属本社は自動セット）
⑤ DX企業名を入力
⑥ 「登録」ボタンをクリック
⑦ SGS RDSにPOST（DX企業CDは自動発番）
⑧ ダイアログが閉じ、ツリーにDX企業が追加される
```

### フロー5: 店舗追加

```
① 左ツリーでDX企業ノードをクリック
② 右パネルにDX企業情報が表示される
③ 「店舗追加」ボタンをクリック
④ 店舗追加ダイアログが開く（DX企業CDは自動セット）
⑤ 顧客CD・サービスCD・権限CDを入力
   ↳ サービスCDはIoT対象（U40/U60/U62/U90/U91/U93）から選択
   ↳ 権限CDは 30（支店管理）/ 31（支店シェア）から選択
⑥ 「登録」ボタンをクリック
⑦ SGS RDS service_group_customers にupsert
⑧ ダイアログが閉じ、ツリーに店舗が追加される
```

### フロー6: UMアカウント作成・紐づけ（本社レベル）

```
① 左ツリーで本社ノードをクリック
② 右パネルのhonshaStoreカードで対象サービスの「UMアカウント作成」ボタンをクリック
   （そのサービス×UNISCDにすでにアカウントがある場合はボタン非活性）
③ UMアカウント作成ダイアログが開く（level = 本社 / 対象サービスCDが自動表示）
④ 認証用メールアドレス・アカウント名を入力
⑤ 「作成・紐づけ」ボタンをクリック
⑥ MEMBERS APIでアカウント作成（or 既存に紐づけ）→ ご利用案内メール自動送信
⑦ SGS RDS service_group_customers にupsert（unis_customer_cd = 本社のUNIS顧客CD、権限CD = 20固定）
⑧ ダイアログが閉じ、UMID一覧が更新される（仮登録バッジで表示）
```

### フロー7: UMアカウント作成・紐づけ（店舗レベル）

フロー6と同様。ただし:
- ツリーで「店舗×サービス行」をクリックして店舗カードを表示してからボタンをクリック
- level = store、unis_customer_cd = 店舗のUNISCD、権限CD = 30固定

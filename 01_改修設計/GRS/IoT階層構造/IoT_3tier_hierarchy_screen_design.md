| 項目 | 内容 |
|------|------|
| 画面名 | IoT 本社 / DX企業 / 店舗管理 |
| 画面ID | GRS-MOCK-IOT-HIERARCHY |
| 対象システム | GRS (usen-grs-mini-web) |
| 作成日 | 2026-03-26 |

---

## 1. 画面概要

IoTサービス（カメラ・WiFi等）の契約階層を管理する運用ツール画面。
**本社 → DX企業 → 店舗** の3階層構造をツリー表示し、各ノードの詳細確認・登録・UMID紐づけを行う。

---

## 3. 画面要素一覧

### 3-1. ヘッダーエリア

| 要素 | 種別 | 表示内容 | 備考 |
|------|------|----------|------|
| 画面タイトル | テキスト | "IoT 本社/企業/支店管理" | h1相当 |
| バッジ | チップ | "API連携" | 青色・常時表示 |
| 本社/DX企業登録ボタン | ボタン（outlined） | "本社 / DX企業登録" | クリックで登録ダイアログを開く（→ §4-1） |

---

### 3-2. 検索フォーム

| 要素 | 種別 | 内容 | 備考 |
|------|------|------|------|
| 検索タブ | `v-btn-toggle`（6タブ） | 下記6種から選択 | タブ切り替え時に入力フィールドを自動クリア |
| 検索キーワード | テキスト入力 | 選択タブに対応したラベル・プレースホルダー | Enterキーでも検索実行、クリアボタンあり |
| 検索ボタン | ボタン（primary） | "検索" | クリックでAPI検索を実行、ローディングスピナー表示 |

**検索タブの選択肢（v-btn-toggle）:**

| タブラベル | アイコン | value | 検索方式 |
|-----------|---------|-------|---------|
| 本社CD | mdi-office-building | company_cd | 完全一致 |
| 本社名 | mdi-office-building-search | company_name | 部分一致（LIKE） |
| DX企業CD | mdi-layers | group_cd | 完全一致 |
| DX企業名 | mdi-text-search | group_name | 部分一致（LIKE） |
| 設置先CD | mdi-store | inst_store_cd | 完全一致 |
| 設置名 | mdi-store-search | store_name | 部分一致（LIKE） |

**表示状態の遷移:**

```
[未検索]           → "{選択中タブ名}を入力して検索してください"（動的テキスト）
[検索中]           → ローディングスピナー
[名前系: 複数ヒット] → 候補一覧（§3-3）
[1件ヒット or CD系] → ツリー＋詳細パネル（§3-4, 3-5）。検索キーに一致したノードが自動ハイライト（選択状態）
[0件]              → "該当データなし" メッセージ
```

**検索後のツリー表示とハイライト:**

- 検索結果は常にフルツリー（本社〜配下全ノード）を表示する
- 検索種別に応じて、検索キーに一致したノードを自動選択・ハイライトする
  - 本社CD / 本社名 → 本社ノードをハイライト
  - DX企業CD / DX企業名 → 該当DX企業ノードをハイライト（親本社は自動展開）
  - 設置先CD / 設置名 → 該当店舗×サービス行をハイライト（親DX企業まで自動展開）
- 候補一覧から選択した場合も同じハイライト動作を行う

### 3-3. 候補一覧（名前検索で複数ヒット時）

名前系（本社名 / DX企業名 / 設置名）で複数の本社がヒットした場合に表示。

| 要素 | 内容 |
|------|------|
| タイトル | "複数の本社が該当しました（N件）" |
| 一覧行 | 本社名（太字）、本社CD、顧客CD を表示。行クリック or「この本社を表示」ボタンで選択 → ツリー表示へ遷移 |

---

### 3-4. ツリーパネル（左側）

| 要素 | 種別 | 内容 |
|------|------|------|
| パネルタイトル | テキスト | "本社 / DX企業 / 店舗" |
| 全展開/全閉じるボタン | テキストボタン | トグルで全ノードの開閉を切替 |
| ツリーノード | 行 | クリックで右パネルに詳細表示（アクティブ強調）、chevronで開閉 |

**ノード種別とスタイル:**

| 階層 | アイコン | インデント | 表示内容 |
|------|---------|-----------|---------|
| 本社 | mdi-office-building（青） | 0 | 本社名（太字・青） |
| DX企業 | mdi-layers（緑） | 32px | DX企業名 |
| 店舗 | mdi-store（グレー） | 64px | 店舗名 + サービスCDチップ（例: "U40 Camera"） |

**店舗ノードの行数:**
1店舗が複数のサービスに紐づく場合、サービスの数だけ行を作成する（1店舗×1サービス = 1行）。同じ店舗が「U40 Camera」「U90 サイネージ」の2つのサービスを持つ場合、その店舗は2行表示される。

---

### 3-5. 詳細パネル（右側）

ツリーで選択したノード種別に応じて3パターンの詳細を表示する。

#### パターンA: 本社ノード選択時

**基本情報カード**

| 要素 | 内容 |
|------|------|
| タイトル | 本社名、"本社" チップ |
| DX企業追加ボタン | "DX企業追加"（→ §4-1 の addGroup モードで登録ダイアログを開く） |
| 顧客CD | UNIS顧客CD（読取専用表示） |
| 本社CD | companiesテーブルのCD（読取専用表示） |
| DX企業一覧 | 配下の全DX企業名をチップで表示（件数つき） |

**本社レベル店舗カード（honshaStores, 0件以上）**

本社に直接紐づく店舗（store_cd='0000'相当）を物理店舗単位で1枚表示。同一店舗に複数サービスが紐づく場合でもカードは1枚。

| 要素 | 内容 |
|------|------|
| タイトル | 店舗名、全サービスCDチップ（複数可）、"本社" チップ |
| 店舗情報セクション | 店舗CD、店舗名、管理用店舗UNISCD |
| サービス期間セクション（サービスCDごと） | 開始日・終了日をサービスCDチップ付きで表示。サービス数分繰り返す |
| USENMEMBERSアカウントヘッダー | 件数バッジ + **UMアカウント作成ボタン**（→ §4-3、level=honsha）。そのサービス×UNISCDにすでにアカウントがある場合はボタン非活性（disabled） |
| アカウント行（0件以上） | 全サービスのUMIDを統合した一覧。UMID・アカウント名・所属サービスCDチップ・**仮登録/本登録バッジ**を表示。行クリックで詳細展開（§3-6） |

---

#### パターンB: DX企業ノード選択時

| 要素 | 内容 |
|------|------|
| タイトル | DX企業名、"DX企業CD" チップ |
| 店舗追加ボタン | "店舗追加"（→ §4-2） |
| DX企業CD | service_group_cd 表示 |
| 所属本社 | 親本社名 表示 |
| 配下店舗テーブル | 下記カラム構成 |

**配下店舗テーブルのカラム:**

| カラム | 内容 | 備考 |
|-------|------|------|
| 店舗名 | 店舗アイコン + 店舗名 | 行クリックでその店舗を詳細表示 |
| サービスCD | サービスCDチップ（複数可） | U40 / U90 / U91 等 |
| UMID数 | 全サービスの合計件数 | |
| （矢印） | mdi-chevron-right | クリック可であることを示す |

---

#### パターンC: 店舗ノード選択時

ツリーで選択した「店舗×サービス」の1サービス分を1枚のカードで表示。

| 要素 | 内容 |
|------|------|
| タイトル | 店舗名、選択中サービスCDチップ（例: "U40 Camera"） |
| サービス期間セクション | 選択中サービスの開始日・終了日 |
| USENMEMBERSアカウントヘッダー | 件数バッジ + **UMアカウント作成ボタン**（→ §4-3、level=store）。そのサービス×UNISCDにすでにアカウントがある場合はボタン非活性（disabled） |
| アカウント行（0件以上） | 選択中サービスのUMID一覧。UMID・アカウント名・**仮登録/本登録バッジ**を表示。行クリックで詳細展開（§3-6） |

---

### 3-6. アカウント行のアコーディオン展開

パターンA・Cのアカウント行をクリックすると、行の下に詳細情報がスライド展開する。再クリックで折りたたむ。

**展開エリアの構成:**

**① 共通情報**

| フィールド | 内容 |
|-----------|------|
| UMID | USENMEMBERSのユーザーID |
| ログインID | MEMBERSログインID |
| アカウント名 | 表示名 |
| 権限 | 本社管理（20）または支店管理（30） |
| 認証用メールアドレス | ログイン認証に使用するメールアドレス |
| 連絡先メールアドレス | 連絡先として登録されたメールアドレス |

**② サービスごとの情報（サービス数分繰り返す、パターンAのみ）**

本社ノードのアカウント行では、同一UMIDが複数サービスに紐づく場合、サービスCDチップをヘッダーとしてサービスごとに区切って表示する。

| フィールド | 内容 |
|-----------|------|
| 利用開始日 | そのサービスの利用開始日 |
| 利用終了日 | そのサービスの利用終了日（未設定時は "-"） |

## 4. ダイアログ

### 4-1. 本社 / DX企業登録ダイアログ

**2つのモードを共用する:**

| モード | 開き方 | 本社情報欄 |
|--------|--------|-----------|
| `new`（新規） | ヘッダーの「本社 / DX企業登録」ボタン | 入力可 |
| `addGroup`（DX企業追加） | 本社詳細の「DX企業追加」ボタン | 読取専用（親本社の値が自動入力） |

**入力フォーム（3セクション）:**

**① 本社情報**

| フィールド | 必須 | 備考 |
|-----------|------|------|
| 管理用UNISCD | ○ | addGroupモードでは読取専用 |
| 本社名（設置先名） | ○ | addGroupモードでは読取専用 |
| サービスCD | ○（newモードのみ） | newモードのみ表示・必須。入力すると `neos_contracts` に既存UMIDがあるか確認し、ある場合はアラートで表示。addGroupモードでは非表示 |

**② DX企業情報**

| フィールド | 必須 | 備考 |
|-----------|------|------|
| DX企業名 | ○ | DX企業CDは自動発番のため入力なし |

**③ 支店情報**

| フィールド | 必須 | 備考 |
|-----------|------|------|
| 顧客CD | ○ | 最初の支店（設置先）のUNISCD |
| サービスCD | ○ | 例: U40 / U91 |

**ボタン:**

| ボタン | 動作 |
|-------|------|
| キャンセル | ダイアログを閉じる |
| 登録実行 | 全必須項目が入力済みの場合のみ活性化。クリックでAPI送信（モック上はスナックバー表示） |

---

### 4-2. 店舗追加ダイアログ

DX企業詳細の「店舗追加」ボタンから開く。
追加先のDX企業名 / DX企業CDは自動表示（読取専用）。

| フィールド | 必須 | 備考 |
|-----------|------|------|
| 追加する店舗の顧客CD | ○ | 設置先のUNISCD |
| サービスCD | ○ | 例: U40 |

**ボタン:**

| ボタン | 動作 |
|-------|------|
| キャンセル | ダイアログを閉じる |
| 追加登録 | 入力済みの場合のみ活性化。クリックでAPI送信 |

---

### 4-3. UMアカウント作成ダイアログ

本社レベル店舗カード / 店舗詳細カードの「UMアカウント作成」ボタンから開く。
呼び出し元によって **level（本社 / 支店）** と **対象サービスCD** がヘッダーに自動表示される。

**統合フロー（新規作成と既存紐づけは分けない）:**

入力したメールアドレスが既存UMIDを持つアカウントの場合は自動で紐づけを行う。
ユーザーはモードを意識せず、メールアドレスとアカウント名を入力するだけでよい。

**入力フィールド:**

| フィールド | 必須 | 備考 |
|-----------|------|------|
| 認証用メールアドレス | ○ | 既存アカウントの場合は自動で紐づけ |
| アカウント名 | ○ | MEMBERSのアカウント表示名 |

**アラート（ダイアログ内）:**

- "入力したメールアドレスが既存アカウントの場合は自動で紐づけを行います。作成後、ご利用案内メールがアカウントに送信されます。"

**ボタン:**

| ボタン | 動作 |
|-------|------|
| キャンセル | ダイアログを閉じる |
| 作成・紐づけ | 両フィールド入力済みの場合のみ活性化 |

## 5. 入出力サマリ

### API Input（画面 → バックエンド）

| 操作 | 主要パラメータ |
|------|-------------|
| 検索 | 検索種別（searchMode）、キーワード（searchQuery） |
| 本社 / DX企業登録（新規） | unis_customer_cd、company_name、service_group_name、shiten_unis_customer_cd、shiten_service_cd |
| DX企業追加 | unis_customer_cd（親本社）、service_group_name、shiten_unis_customer_cd、shiten_service_cd |
| 店舗追加 | service_group_cd（親DX企業）、unis_customer_cd、service_cd |
| UMアカウント作成・紐づけ | email、username、level、service_cd |

### API Output（バックエンド → 画面）

| データ | 画面での用途 |
|--------|------------|
| 本社一覧（company[]） | 検索結果の候補一覧 / ツリー表示 |
| 本社詳細 | 顧客CD、本社CD、honshaStores、配下DX企業一覧 |
| DX企業詳細 | DX企業CD、所属本社名、配下店舗一覧 |
| 店舗詳細 | サービスCD、開始/終了日、紐づきUMID一覧 |
| UMID情報 | umid、umembers_grant_cd、紐づき済みフラグ |

## 6. 状態・バリデーション

| 条件 | 挙動 |
|------|------|
| 検索キーワード空で検索 | API未実行（バリデーション or 空結果） |
| 登録フォームの必須項目未入力 | 「登録実行」ボタンを非活性化 |
| 店舗追加の必須項目未入力 | 「追加登録」ボタンを非活性化 |
| UMID入力不足 | 「作成・紐づけ / 紐づけ実行」ボタンを非活性化 |
| 登録・追加成功 | スナックバー（画面右下）に成功メッセージを4秒表示 |

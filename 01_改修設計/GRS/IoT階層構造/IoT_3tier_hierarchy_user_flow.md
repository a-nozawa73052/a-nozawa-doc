## F-1 本社・DX企業・店舗を一括新規登録

**想定シナリオ:** まだシステムに存在しない企業グループを一から登録したい。

```
[IoT管理画面]
    │
    ▼
①「本社 / DX企業登録」ボタンをクリック
    │
    ▼
② 登録ダイアログが開く（new モード）
    │
    ├─ [本社情報] 管理用UNISCD *      ← 入力
    │              本社名（設置先名）*  ← 入力
    │
    ├─ [DX企業情報] DX企業名 *        ← 入力
    │               ※DX企業CDは自動発番（入力不要）
    │
    └─ [支店情報]  顧客CD *           ← 入力（最初の支店のUNISCD）
                   サービスCD *        ← 入力（例: U91）
    │
    ▼
③「登録実行」ボタンをクリック（全必須項目入力後に活性化）
    │
    ▼
④ 登録成功 → ダイアログが閉じる → スナックバー表示
    │
    ▼
⑤ 検索フォームに新しい本社CDを入力して検索 → ツリーで確認
```

**登録により作成されるデータ:**
- 本社レコード（companies）
- DX企業レコード（service_groups相当）
- 最初の支店レコード（service_group_customers）

## F-2 既存本社にDX企業を追加

**想定シナリオ:** すでに本社は登録済みで、新しいブランド（DX企業）を追加したい。

```
[IoT管理画面]
    │
    ▼
① 検索フォームで対象の本社を検索
    │  例）検索項目: 本社CD、値: PD000012264
    │
    ▼
② ツリーパネルに本社が表示される
    │
    ▼
③ ツリーの「本社ノード」をクリック → 右パネルに本社詳細が表示
    │
    ▼
④ 本社詳細カードの「DX企業追加」ボタンをクリック
    │
    ▼
⑤ 登録ダイアログが開く（addGroup モード）
    │
    ├─ [本社情報] 管理用UNISCD       ← 自動入力・読取専用
    │              本社名              ← 自動入力・読取専用
    │
    ├─ [DX企業情報] DX企業名 *        ← 入力
    │
    └─ [支店情報]  顧客CD *           ← 入力
                   サービスCD *        ← 入力
    │
    ▼
⑥「登録実行」ボタンをクリック
    │
    ▼
⑦ 登録成功 → スナックバー表示
    │
    ▼
⑧ 再検索してツリーに新DX企業が追加されていることを確認
```

**F-1との違い:** 本社情報は読取専用で表示済み。DX企業名と支店情報のみ入力する。

## F-3 既存DX企業に店舗を追加

**想定シナリオ:** すでにDX企業は存在し、新店舗がオープンしたので追加したい。

```
[IoT管理画面]
    │
    ▼
① 検索フォームで対象を検索
    │  例）検索項目: DX企業CD、値: 009WK
    │  　　または: 本社名で検索 → 候補一覧から選択
    │
    ▼
② ツリーに本社 + DX企業が表示される
    │
    ▼
③ ツリーの「DX企業ノード」をクリック → 右パネルにDX企業詳細が表示
    │  配下の店舗一覧テーブルが確認できる
    │
    ▼
④「店舗追加」ボタンをクリック
    │
    ▼
⑤ 店舗追加ダイアログが開く
    │  追加先DX企業名/CDが自動表示（読取専用）
    │
    ├─ 追加する店舗の顧客CD *  ← 入力
    └─ サービスCD *            ← 入力（例: U40）
    │
    ▼
⑥「追加登録」ボタンをクリック（両項目入力後に活性化）
    │
    ▼
⑦ 登録成功 → スナックバー表示
    │
    ▼
⑧ DX企業詳細の店舗テーブルに新しい店舗が表示されることを確認
```

## F-4 本社レベルのUMIDを作成・紐づけ

**想定シナリオ:** 本社で管理するIoTサービスのUMアカウントを作成したい。

```
[IoT管理画面]
    │
    ▼
① 検索フォームで対象の本社を検索
    │
    ▼
② ツリーの「本社ノード」をクリック → 本社詳細を表示
    │
    ▼
③ honshaStoreカード（本社レベルの店舗）を確認
    │  全サービス統合のUMID一覧が表示されている（サービスCDチップ + 仮登録/本登録バッジ + 権限チップ付き）
    │  既存UMIDの行をクリックするとアコーディオンで詳細確認も可能
    │
    ▼
④ サービスごとの「UMアカウント作成」ボタンをクリック（アカウントがある場合はボタン非活性）
    │
    ▼
⑤ UMアカウント作成ダイアログが開く（level = 本社）
    │
    ├─ 認証用メールアドレス *  ← 入力
    └─ アカウント名 *          ← 入力（MEMBERSのアカウント表示名）
    │
    ▼
⑥「作成・紐づけ」ボタンをクリック
    │
    ▼
⑦ 成功 → スナックバー表示。作成後、ご利用案内メールがアカウントに自動送信される
    │
    ▼
⑧ honshaStoreカードのUMID一覧に新しいアカウントが「仮登録」で追加される
```

## F-5 店舗レベルのUMIDを作成・紐づけ

**想定シナリオ:** 特定の店舗のIoTサービス（カメラ等）のUMアカウントを作成したい。

```
[IoT管理画面]
    │
    ▼
① 検索フォームで対象を検索
    │  例）検索タブ: 設置名、値: "渋谷店"
    │  　　→ 候補一覧から該当本社を選択 → 該当店舗行が自動ハイライト
    │
    ▼
② ツリーで該当の「店舗×サービス行」をクリック（例: "渋谷店 / U40 Camera"）
    │
    ▼
③ 右パネルに選択サービスの店舗カードが表示される
    │  対象サービスの利用期間と、そのサービスのUMID一覧が表示される（仮登録/本登録バッジ付き）
    │
    ▼
④「UMアカウント作成」ボタンをクリック（アカウントがある場合はボタン非活性）
    │
    ▼
⑤ UMアカウント作成ダイアログが開く（level = 支店）
    │
    ├─ 認証用メールアドレス *  ← 入力
    └─ アカウント名 *          ← 入力
    │
    ▼
⑥「作成・紐づけ」ボタンをクリック
    │
    ▼
⑦ 成功 → スナックバー表示。作成後、ご利用案内メールがアカウントに自動送信される
```

## F-6 既存UMIDを既存店舗に紐づけ

**想定シナリオ:** すでにUSENMEMBERSアカウントを持っているユーザーを、店舗のIoTサービスに紐づけたい。

```
[IoT管理画面]
    │
    ▼
① F-4 または F-5 と同じ手順で、対象の本社/店舗カードを表示
    │
    ▼
②「UMアカウント作成」ボタンをクリック
    │
    ▼
③ UMアカウント作成ダイアログが開く
    │
    ├─ 認証用メールアドレス *  ← 既存アカウントのメールアドレスを入力
    └─ アカウント名 *          ← 入力
    │
    ▼
④「作成・紐づけ」ボタンをクリック
    │  ※ 入力したメールアドレスが既存UMIDを持つ場合、
    │     新規作成せず自動で既存アカウントに紐づけ
    │
    ▼
⑤ 成功 → スナックバー表示
```

**F-5との違い:** ユーザーから見た操作は同じ。入力したメールアドレスに既存UMIDがあるかどうかをバックエンドが判断し、新規作成か既存紐づけかを自動で切り替える。

## F-7 特定の企業・店舗を検索して確認する

**想定シナリオ:** 登録内容の確認や、サポート対応時に特定の企業・店舗を素早く見つけたい。

```
[IoT管理画面]
    │
    ▼
① 検索タブ（v-btn-toggle）で検索種別を選ぶ
    │
    ├─ CDがわかっている場合 ─────────────────────────────────────────┐
    │  （本社CD / DX企業CD / 設置先CD）                                │
    │   → キーワード入力 → 検索                                       │
    │   → 1件ヒット → 直接ツリー表示                                  │
    │                                                                  │
    └─ 名前しかわからない場合 ─────────────────────────────────────┐  │
       （本社名 / DX企業名 / 設置名）                                │  │
        → キーワード入力 → 検索                                     │  │
        → 複数ヒット → 候補一覧が表示される                         │  │
            → 「この本社を表示」をクリックして選択                  │  │
        → 1件ヒット → 直接ツリー表示                               │  │
                                                                    ↓  ↓
    ② ツリーパネルに本社 + 配下DX企業 + 店舗が表示される
        │  検索キーに一致したノードが自動ハイライト（選択状態）される
        │
        ▼
    ③ ツリーのノードをクリック → 右パネルに詳細が表示される
        │
        ├─ 本社ノード選択   → 本社情報カード + honshaStoreカード
        ├─ DX企業ノード選択 → DX企業情報カード + 配下店舗テーブル
        └─ 店舗×サービス行選択 → 選択サービスのカード（サービス期間＋UMID一覧付き）
        │
        ▼
    ④ 必要に応じて各カードのボタンから後続フローへ（F-2〜F-6）
```
